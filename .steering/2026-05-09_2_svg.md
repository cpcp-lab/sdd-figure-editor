# SVG ファイル入出力

## 要求事項

- キャンバス上の図形を SVG ファイルとして保存できる
- 保存した SVG ファイルを読み込んでキャンバスに再現できる
- SVGのサブセットをサポートする
- メニューバーを追加し，ファイルメニューとヘルプメニューを備える

## 達成基準

- エディタで描いた図形 (Line，Rectangle，Circle，Ellipse，Polyline，Polygon)
  を SVG ファイルに書き出し，ブラウザや SVG ビューアで正しく表示されること
- 書き出した SVG ファイルを読み込み，元の図形が再現されること
- JUnit テストで SVG 書き出し・読み込みのラウンドトリップを検証すること
- W3C SVG 1.1 テストスイートの shapes-*.svg のうち対応範囲内のファイルを読み込み，
  対応する参照 PNG (`W3C_SVG_11_TestSuite/png/`) との画素比較テストがパスすること
  - 比較方法: 差分ピクセル率が 10% 以下を合格基準とする
    (テキスト未描画・アンチエイリアス差異を考慮して 10% を採用)
  - テスト対象ファイル (15 件):

    | ファイル | 備考 |
    |---------|------|
    | shapes-circle-01-t, 02-t | |
    | shapes-ellipse-01-t, 02-t | |
    | shapes-intro-01-t | |
    | shapes-line-01-t | |
    | shapes-polygon-01-t, 02-t, 03-t | |
    | shapes-polyline-01-t | |
    | shapes-rect-01-t, 02-t | |
    | shapes-rect-04-f, 06-f, 07-f | -f だが図形は対応可能 |

  - 除外 (対応困難): shapes-rect-03-t (scale transform，`<use>`)，
    shapes-ellipse-03-f，shapes-line-02-f，shapes-rect-05-f (rotate/skew transform)，
    shapes-grammar-01-f (points の特殊フォーマット)
  - 除外 (`<path>` 要素を含む): shapes-polyline-02-t，shapes-intro-02-f

## 方針

- SVG の書き出しは各 `Figure` サブクラスに `toSvg() : String` メソッドを追加して実装する
- SVG の読み込みは標準 XML パーサ (javax.xml) を使用して実装する
- `EditorFrame` にメニューバー (File > Save / Open) を追加する
- SVG 読み込みで対応する機能:
  - 図形要素: `<circle>`, `<ellipse>`, `<line>`, `<rect>`, `<polyline>`, `<polygon>`
  - スタイル属性: `fill`, `stroke`, `stroke-width` (named color，`#RGB`/`#RRGGBB`，`none`)
  - 属性省略時のデフォルト値: `cx`/`cy` → 0，`x`/`y` → 0 など SVG 仕様に従う
  - `<g>` 要素の `transform="translate(x, y)"` と `fill`/`stroke` の子要素への継承
  - `<rect>` の角丸属性 `rx`/`ry`
  - 不明要素 (`<text>`, `<defs>`, `<use>` など) はスキップ
- SVG 読み込みテストは `SvgReader` で Figure リストを生成し，480×360 の `BufferedImage` に
  レンダリングして参照 PNG と画素比較する
  - レンダリングエンジンの差異 (アンチエイリアシング等) を考慮し，差分ピクセル率 5% 以内を合格基準とする
